# 🛍️ 商城平台治理后台 (Mall Governance Admin)

[![Vue](https://img.shields.io/badge/Vue-3.4+-brightgreen.svg?logo=vuedotjs)](https://vuejs.org/)
[![Vite](https://img.shields.io/badge/Vite-5.4+-blue.svg?logo=vite)](https://vitejs.dev/)
[![Ant Design Vue](https://img.shields.io/badge/Ant%20Design%20Vue-4.2+-blue.svg?logo=antdesign)](https://antdv.com/)
[![TypeScript](https://img.shields.io/badge/TypeScript-5.5+-blue.svg?logo=typescript)](https://www.typescriptlang.org/)

商城平台治理后台是一个独立于商户后台的业务核心管理端。它专为平台的运营、风控、财务及各级管理员设计，提供全局视角下的商品、订单、资金、风险的实时监控和仲裁治理能力。

本项目不仅在**技术栈与视觉体系**上与商户后台保持一致，以降低多端开发与维护成本，同时在功能上打通了双端业务协作闭环（如商户提审与平台审核、买卖纠纷与平台介入仲裁）。

## 🛠️ 核心技术栈

本项目基于现代化的前端生态标准构建，确保了流畅的开发体验与优异的应用性能：

- **核心视图框架**: [Vue 3](https://vuejs.org/) (全面采用 Composition API 与 `<script setup>` 语法)
- **前端构建工具**: [Vite 5](https://vitejs.dev/) (提供极致的冷启动速度与极速的 HMR 热更新)
- **强类型语言**: [TypeScript 5](https://www.typescriptlang.org/) (提供企业级的可靠性与编辑器级别的类型提示)
- **企业级 UI 组件库**: [Ant Design Vue 4](https://www.antdv.com/) (开箱即用的优质后台组件群)
- **路由状态管理**: [Vue Router 4](https://router.vuejs.org/)
- **网络请求客户端**: [Axios](https://axios-http.com/)
- **数据可视化体系**: [G2Plot](https://g2plot.antv.vision/) (按需加载的高交互型数据分析图表)
- **工程化与团队规范**:
  - `ESLint` + `Prettier` 保障团队开发的代码质量与风格统一。
  - `Husky` + `Lint-staged` 实现 Git Code Review 前的自动化安全检查。

## 🚀 核心业务模块与全景图

系统采用强模块化设计思想，已完整规划并覆盖了商城平台治理场景的 12 大垂直业务域：

1. 📊 **全局仪表盘 (Dashboard)**: T+0 级治理信号灯预警雷达、平台各线待办工单聚合面板。
2. 🛍️ **商品治理中心**: 面向全量商品的瀑布流管控、机器辅助人工审核流、特殊品牌准入壁垒、敏感类目红线管控。
3. 📦 **订单治理中心**: 履约异常订单智能监控池、买卖双方争议订单的判责引擎、深度特征详情全景追溯。
4. ⚖️ **售后仲裁法庭**: 买卖举证材料质检管理、平台介入后的中立仲裁法槌、平台判责案例知识库。
5. 🚚 **发货与物流引擎**: 快递轨迹异常停滞打标、商户运费模板跨区成本审核、疑难包裹下行追索。
6. 👔 **商户协同管理**: 商家经营状态分层评级、全维履约健康度画像分析、平台下发必治整改任务验收闭环。
7. 💰 **财务结算中心**: 精细化多维度结算单出账、合规发票核销及验真通道、平台-物流商-商户三边总账对账引擎。
8. 🎯 **营销投入治理**: 促销活动申报资质准入、高额优惠券预算消耗漏斗监控、劣质核销阻断策略及全局 ROI 核算报表。
9. 🛡️ **风控安全中心**: 动态风险策略规则集、实时高危行为命中档案、限制销售/降权/下架等梯度处罚执行模块与申诉绿道。
10. ⚙️ **系统与权限引擎**: 基于标准的 RBAC 模型设计、支持按钮级与接口级的细粒度越权防护、灵活可配的平台全局参数字典表。
11. 🔌 **开放生态平台**: 支持与商家自研系统打通的 Webhook 事件总线机制、租户级 API Key 凭证签发与速率强制限流。

### 🔄 双端业务深度协同机制 (Platform ↔ Merchant)

本系统在底层设计上，着重体现了平台裁判员与商户运动员间的高效协作信息闭环模型：

- **商品审核流协同**: `商家后台提交流水线` -> `平台命中机审质检` -> `(若需)触发人工流转审核` -> `驳回补件指令` -> `商户重提交修正` -> `平台二次复审通过即刻上架`。
- **订单履约异常协同**: `系统跑批拦截异常订单流水` -> `平台生成跨端协同工单下发` -> `商户通过工作台查询并回传证明文件` -> `平台审核运营确认凭证后手动解封或进入强裁决环节`。
- **全局通知统一联动**: 系统产生的各类**确权及状态变更**（商品被驳回、争议订单判责完成、异步大文件导出任务结束）等，均由底层的**统一消息中心 (Message Center)** 封装投递，并跨终端实时推送到商户的运营系统。

## 🏃 快速上手指南

### 1. 开发环境建议准备

为了保证开发体验与依赖库兼容性，请确保您的本地计算机已安装了以下基础环境：
- Node.js (推荐 v18.x(LTS) 或 v20.x+)
- npm (通常随 Node.js 一并安装，可兼容 yarn / pnpm)
- Git (版本控制与提交)

### 2. 项目初始化与服务启动

只需四步极简命令即可在本地将后台系统运行起来：

```bash
# 第一步：克隆远端代码仓库至本地文件夹
git clone <repository_url>

# 第二步：进入已拉取的项目根目录
cd manager-web

# 第三步：安装完整的层级依赖包图谱
npm install

# 第四步：唤醒本地开发服务器
npm run dev
```

启动命令执行完毕后，控制台将输出本地 URL（默认监听：**[http://localhost:5180](http://localhost:5180)**）。浏览器访问该地址即可加载系统。

## ⌨️ 可用的构建脚本 (Scripts 命令集)

项目的 `package.json` 内置了一系列工程化 NPM Scripts：

| 控制台输入指令 | 脚本职责与说明 |
| :--- | :--- |
| `npm run dev` | 以开发模式启动高速的 Vite 代理服务器，将支持毫秒级的无刷新热模块重载（HMR）。 |
| `npm run build` | **生产构建流**：底层执行生产环境的 JS/CSS Tree Shaking 和深度资源压缩混淆，构建前会强制调用 `typecheck` 。 |
| `npm run preview` | 本地预览服务引擎：基于 `dist` 产出目录启动本地服务器，可用来提前模拟评估生产环境中的项目真实运行表现状态。 |
| `npm run typecheck` | 仅调用 TypeScript 编译器 (`vue-tsc`) 执行全项目无产物的静态类型语法纠错与推导检查分析。 |
| `npm run lint` | 调用 ESLint + 插件全家桶扫描并检测项目中的 Vue 与 TS 代码是否符合企业既定的规约规范与防范潜在逻辑漏洞风险。 |
| `npm run lint:fix` | 更进一步，除了代码扫描还可以通过 ESLint 规约库自动尝试修复解决简单的格式问题或不统一风格隐患。 |
| `npm run format` | 针对全局，使用 Prettier 的强制排版引擎，将项目的空格缩进、换行彻底打平为统一风貌输出版本。 |
| `npm run prepare` | 生命周期钩子工具（由 Husky 占用），用于开发人员在执行完 `npm install` 后，自动为当前版本库挂载并配置本地 Git Hooks 防护网。 |

## 💡 开发约定与现状说明 (必读)

1. **“设计与 UI 分形优先” 交付策略**:
   鉴于管理系统拥有极其庞大的页面枝节体量，目前 `0.1.0` 版本的核心工程目标是：系统全面完成**前端统一治理架构的搭建落地**及**各高频次业务链路核心页原型的视觉开发工作**。
   因此，一些暂时次要的二级页、分析看板等路由暂时统一配置并指向了 `<ModulePlaceholder />` 开发建设中占位组件，在日后的迭代长跑下可根据需求排期持续进行组件抽离、接口替换对调和深层次的渲染重构优化。

2. **高内聚的鉴权与路由级守卫控制**:
   通过 Vue Router 的能力，已经实现了成熟的前端路由前置拦截器链路 (Navigation Guards)。进入系统前将无死角检查请求主体的 `Token` 有效状态与超时状态机制，将无效请求强行重定向遣返回 `/login` 。

3. **开箱即有的主题外观驱动 (Dark Mode Design System)**:
   样式设计语言层面采用统一的自定义 Native CSS 变量技术实现。不光通过对 UI 库样板的极佳覆写带来企业级定制质感，还使得整套代码骨架自带原生的「多主题变装」能力支撑点 (配合 `[data-theme='dark']` 方案) ，随时准备拥抱深色宇宙模式。
